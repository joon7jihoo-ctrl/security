## 1. 프로젝트 개요

### 1.1 배경 및 목적

KDN 정보보안팀은 보안/컴퓨터공학 전공 신입직원이 실무 투입 전 보안관제 인프라 전반을
빠르게 이해할 수 있는 온보딩 문서의 부재를 인식하였다.

기존 문서의 문제점:
- **단순 개념 나열** 방식으로 실무 적용 어려움
- **장비 간 유기적 관계** 설명 부재
- 가독성 낮은 텍스트 중심 구성으로 **학습 효율 저하**

이에 따라 실제 트래픽 흐름과 장비 간 상호작용을 기술적으로 기술하고,
실무에 즉시 활용 가능한 수준의 보안관제 온보딩 문서를 웹 기반으로 개발하였다.

### 1.2 개발 범위

| 구분 | 내용 |
|------|------|
| **대상 장비** | DDoS 방어 / 방화벽 / IPS / WAF / 가시화 장비 / SIEM·SOAR (eyecloudxoar) |
| **계층 범위** | OSI L2 ~ L7 전 계층 |
| **배포 방식** | GitHub Pages (웹 접근) |
| **형식** | 단일 파일 SPA (HTML + CSS + JS 인라인) |

## 3. 기술 구현 평가

### 3.1 아키텍처 선택 적정성

#### 단일 파일 SPA 방식 채택 근거

```
장점
├── 외부 서버·빌드 툴 불필요 → 즉시 배포 가능
├── 파일 1개만 관리 → 유지보수 단순
├── 오프라인 환경 (내부망) 에서도 열람 가능
└── 이메일 첨부·USB 전달도 가능

단점 (인식된 트레이드오프)
├── 파일이 커질수록 편집 복잡도 증가 (현재 1,176줄)
├── 컴포넌트 재사용 어려움 (복붙 방식)
└── 향후 대규모 확장 시 React 마이그레이션 필요
```

**결론:** 초기 버전(v1.0)에서 콘텐츠 양과 팀 역량을 고려할 때 적절한 선택.
v2.0에서 Vite + React 마이그레이션 권장.

### 3.3 JavaScript 구현 평가

```javascript
// 구현된 기능 (23줄)

  // 현재 뷰포트 섹션 → 사이드바 active 클래스 동기화
```

- **범위:** 사이드바 스크롤 연동 단일 기능

- **평가:** 목적에 부합하는 최소한의 구현. 기능 확장 시 구조화 필요.
